def plot_elbow_curve(X, K):
    # scipy.cluster.vq.kmeans
    KM = [kmeans(X,k) for k in K]
    centroids = [cent for (cent,var) in KM]   # cluster centroids
    avgWithinSS = [var for (cent,var) in KM] # mean within-cluster sum of squares

    # alternative: scipy.cluster.vq.vq
    #Z = [vq(X,cent) for cent in centroids]
    #avgWithinSS = [sum(dist)/X.shape[0] for (cIdx,dist) in Z]

    # alternative: scipy.spatial.distance.cdist
#     D_k = [cdist(X, cent, 'euclidean') for cent in centroids]
#     cIdx = [np.argmin(D,axis=1) for D in D_k]
#     dist = [np.min(D,axis=1) for D in D_k]
#     avgWithinSS = [sum(d)/X.shape[0] for d in dist]

    ##### plot ###
    kIdx = 3
    # elbow curve
    fig = plt.figure(figsize=(15,5))
    ax = fig.add_subplot(111)
    ax.plot(K, avgWithinSS, 'b*-')
    ax.plot(K[kIdx], avgWithinSS[kIdx], marker='o', markersize=12, 
        markeredgewidth=2, markeredgecolor='r', markerfacecolor='None')
    plt.grid(True)
    plt.xlabel('Number of clusters')
    plt.ylabel('Average within-cluster sum of squares')
    plt.title('Elbow for KMeans clustering')
    plt.show()

Características do KMeans¶

De maneira geral, algoritmos de cluster n√£o conseguem lidar t√£o bem com vari√°veis categ√≥ricas, pois √© dif√≠cil achar uma fun√ß√£o de dist√¢ncia que consiga represent√°-las.

Ap√≥s pesquisas realizadas, identificou-se duas formas principais de lidar com essa situa√ß√£o:

  2. Implementação do KMeans¶

Um dos grandes desafios de implentar um algoritmo de clusteriza√ß√£o √© determinar qual o n√∫mero ideal de clusters a ser montados. Ap√≥s pesquisas realizadas, identificou-se duas formas principais de lidar com essa situa√ß√£o:

O gr√°fico de silhueta exibe uma medida de qu√£o pr√≥ximo cada ponto em um cluster √© para pontos nos clusters vizinhos e, portanto, fornece uma maneira de avaliar visualmente os par√¢metros como n√∫mero de clusters. Uma medida num√©rica utilizada √© a average silhouette, que √© a m√©dia dos valores das silhuetas dos clusters. Quanto maior o valor da average silhouette melhor o cluster.

  2. Elbow method [8][9]: este m√©todo analisa a porcentagem de vari√¢ncia explicada como uma fun√ß√£o do n√∫mero de clusters: √© preciso escolher uma s√©rie de clusters para que a adi√ß√£o de outro cluster n√£o ofere√ßa muito melhor modelagem dos dados. Mais precisamente, se uma parcela a porcentagem de vari√¢ncia explicada pelos clusters em rela√ß√£o ao n√∫mero de clusters, os primeiros clusters adicionar√£o muita informa√ß√£o (explicam muita vari√¢ncia), mas em algum momento o ganho marginal cair√°, dando um √¢ngulo na gr√°fico.

Neste estudo utilizaremos o segundo m√©todo para identifica√ß√£o do melhor valor de k para o KMeans. A partir da implementa√ß√£o abaixo, √© poss√≠vel ver que os melhores valores para K s√£o 4 e 5. def evaluate_arima_model(X, arima_order):
    size = int(len(X) * 0.70)
    train, test = X[0:size], X[size:len(X)]
    history = [x for x in train]
    predictions = list()
    for t in range(len(test)):
        model = ARIMA(history, order=arima_order)
        model_fit = model.fit(disp=0)
        output = model_fit.forecast()
        yhat = output[0]
        predictions.append(yhat)
        obs = test[t]
        history.append(obs)
#         print('predicted=%f, expected=%f' % (yhat, obs))
    mse = mean_squared_error(test, predictions)
    rmse = math.sqrt(mse)
    return(rmse)

def evaluate_models(dataset, p_values, d_values, q_values):
    dataset = dataset.astype('float32')
    best_score, best_cfg = float("inf"), None
    for p in p_values:
        for d in d_values:
            for q in q_values:
                order = (p,d,q)
                try:
                    mse = evaluate_arima_model(dataset, order)
                    if mse < best_score:
                        best_score, best_cfg = mse, order
                    print('ARIMA%s RMSE=%.3f' % (order,mse))
                except NameError as e:
                    print e
                    continue
                except ValueError as e:
                    print e
                    continue
                except:
                    print "Unexpected error:", sys.exc_info()[0]
                    continue
    print('Best ARIMA%s RMSE=%.3f' % (best_cfg, best_score))
    return(best_cfg)

Conclusões¶

Neste projeto foram implementados dois modelos principais para estudo dos dados: um KMeans cluster que agrupou os protudos de acordo com suas caracter√≠sticas e um ARIMA Model que previu as vendas de cada um dos produtos.

Foi utilizada o elbow method para definir o melhor valor de K para o KMeans e foi feito um estudo sobre as principais caracter√≠sticas de cada um dos clusters. Em especial existe um cluster que possui um elevado √≠ndice de cancelamentos nas compras. O ARIMA foi validado usando RMSE como m√©trica de avalia√ß√£o e seus par√¢metros foram otimizados usando Grid Search.
